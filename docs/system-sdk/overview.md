# System SDK

Rules systems declare:

- `system.manifest.json`
- actor JSON schema
- item JSON schema
- sheet registrations
- dice formula registrations
- character creation templates
- character origin catalogs and create-time origin application where supported by the rules runtime
- character import normalizers
- compendium entries for system-specific items, spells, talents, gear, clues, rituals, and conditions
- condition and compendium action automation for actor quick rolls
- action-use consumption for spell slots, resources, strain, and consumable quantities
- equipment purchasing and currency updates where supported by the rules runtime
- guided advancement options for leveling or rank progression
- rest and recovery automation for pools, resources, spell slots, and rest-clearing conditions
- encounter threat catalogs and budgeted encounter planning
- monster actor data builders and action quick rolls where supported by the rules runtime
- optional server hooks

See `plugins/example-system-generic-fantasy`, `plugins/example-system-stellar-frontiers`, and `plugins/example-system-mystic-noir` for minimal installable systems. The primary built-in rules runtime is `dnd-5e-srd`, targeting SRD 5.2.1 / 5.5e-compatible play with Fighter, Cleric, and Wizard templates, SRD character origin catalogs, SRD-oriented character import, level advancement, hit dice, spell slots, saving throw quick rolls, class save proficiencies, skill and tool check quick rolls, class/template/background skill and tool proficiencies, SRD currency and equipment purchasing, rest recovery, SRD monster stat blocks with CR/XP values, monster actor data derived from threat stat blocks, monster action quick rolls, 2024 Basic Rules XP encounter budgets, and compendium-backed Longsword, Dagger, Quarterstaff, Shortbow, Spear, Shield gear, Calligrapher's Supplies, Thieves' Tools, Healing Word, Fire Bolt, Cure Wounds, Shield spell, Chromatic Orb, Ice Knife, Ray of Sickness, Alert, Magic Initiate, Savage Attacker, Blessed, Poisoned, and Restrained entries. Longsword, Dagger, Quarterstaff, Shortbow, Spear, Healing Word, Fire Bolt, Cure Wounds, Shield, Chromatic Orb, Ice Knife, Ray of Sickness, and supported monster stat block actions produce actor-aware action formulas when attached to a sheet; SRD Healing Word, Cure Wounds, Chromatic Orb, Ray of Sickness, and Ice Knife secondary damage also support `spellSlotLevel` upcasting formulas and matching spell-slot consumption. The older Generic Fantasy, Stellar Frontiers, and Mystic Noir runtimes remain available as demo systems for later expansion.

API callers can install a campaign's active system, list character templates, list D&D SRD character origin choices, build actors from those templates with starting items and currency, apply D&D SRD background/species selections during character creation, import normalized character data, create D&D SRD monster actors from threat stat blocks, list a system compendium, add item/spell/gear/talent/clue/ritual entries to actors, purchase SRD equipment with actor currency, apply or remove conditions, advance actors through the system's guided progression, apply short or long rest recovery, plan encounters from system threat budgets, optionally persist planned encounters, and retrieve actor sheets that include condition-aware ability/aptitude/skill quick rolls, SRD saving throw quick rolls with class proficiency bonuses, SRD skill and tool checks with proficiency/expertise bonuses, SRD monster action quick rolls, compendium-backed action formulas, and the system's inventory, spell, talent, clue, ritual, and resource surfaces. D&D SRD origin choices include Acolyte, Criminal, Sage, and Soldier backgrounds plus Dragonborn, Dwarf, Elf, Gnome, Goliath, Halfling, Human, Orc, and Tiefling species metadata. Create-time origin application sets background skills/tools, origin feat tracking, species traits/resources, starting 50 GP, and validated SRD ability-score increases. D&D SRD encounter planning now returns monster CR, XP, AC, HP, action summaries, party XP budgets, and a difficulty result based on low/standard/hard thresholds. Monster actor sheets expose attack and damage quick rolls such as `monster-scimitar-attack` and `monster-scimitar-damage` from stat block actions. The system purchase endpoint spends actor currency atomically with item creation and rejects insufficient funds without mutating the actor. The system rest endpoint updates HP/strain/composure, hit dice, resource pools, spell slots where applicable, and rest-clearing conditions. The system roll endpoint can post action formulas into chat by roll id, including SRD `monster-*`, `spell-*`, `item-*`, `skill-*`, `tool-*`, and `save-*` roll ids. `spellSlotLevel` can upcast supported leveled spell healing and damage formulas, `consumeResources: true` spends the matching spell slot, actor resource, strain, or consumable quantity before the roll is posted, and `applyEffect: true` applies supported damage/healing totals to target actor pools with target edit permission.
