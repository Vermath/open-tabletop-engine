# v0.3 UI Dogfood Pass

Date: 2026-05-12

## Scope

End-to-end browser dogfood pass against the local v0.3 app with visual inspection, DOM snapshots, console inspection, network request inspection, and screenshots.

Browser Use note: the Browser Use Node REPL backend was not exposed in this session after tool discovery, so this pass used the Playwright CLI fallback against a real headed browser session named `otte-dogfood`.

## Environment

- API: existing local dev API on `http://localhost:4000`
- Web: `pnpm --filter @open-tabletop/web dev` on `http://localhost:5173`
- Browser viewport checks: `1440x1000` desktop, `390x844` mobile

## Fixes From This Pass

- Kept the desktop tabletop shell inside the viewport with internal rail, inspector, and console scrolling.
- Restored mobile access to campaign/session/export/import controls instead of hiding the rail entirely.
- Wrapped inspector tabs across rows so Actors, Journal, Combat, Content, AI, and SDK are all visible without a horizontal tab scrollbar.

## Remaining UX Notes

- Fog preset save/apply/delete, fog history, and vision sample still use native browser prompt/confirm/alert dialogs. They are functional, but should become in-app dialogs in a later polish pass.
- The mobile rail is intentionally shown before the table so all setup/session controls remain reachable; a future compact mobile drawer would make this denser.
