# Run Your First Session Checklist

## Before Players Join

- [ ] API and web are running.
- [ ] `docs/demo/ember-vault-beta-dogfood.ottx.json` is imported.
- [ ] The campaign appears in the sidebar.
- [ ] Current scene, tokens, journals, and combat tracker are visible to the GM.
- [ ] A backup archive has been exported.

## With Players

- [ ] Three player seats can join or switch in.
- [ ] Each player can see their public scene content.
- [ ] Each player can move an owned token.
- [ ] Dice, chat, journal updates, and combat start appear without manual reload.
- [ ] GM-only notes and hidden tokens stay hidden.

## After The Session

- [ ] Export a campaign archive.
- [ ] Export a Report Bundle if there was any issue.
- [ ] File a v0.3 outside dogfood issue for blockers.
- [ ] Record whether play can continue next session.
