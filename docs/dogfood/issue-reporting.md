# Outside Dogfood Issue Reporting

Use `.github/ISSUE_TEMPLATE/v0.3-outside-dogfood-report.yml`.

## Attach

- Redacted Report Bundle from the sidebar.
- Screenshot or short screen recording when the issue is visual.
- Browser console error text when the web client fails.
- API response status and route when an API action fails.
- Exported `.ottx.json` only when it contains no private table content or the reporter is comfortable sharing it.

## Do Not Attach

- Passwords, session tokens, reset tokens, invite tokens, OAuth details, private email addresses, or proprietary content.
- Full journal, handout, chat, AI, or content-import bodies unless the reporter explicitly wants them public.

## Reproduction Quality

Good reports name the campaign id, user/role, browser, API/web commit, exact steps, expected result, actual result, and whether a refresh or restart recovered the table.
